## ✏️ 一、Commit Message 格式

每次提交請依以下格式撰寫：

```
[類型] 變更摘要

1. 詳細變更項目
2. 詳細變更項目 (點列部分 有許多細項時才需要細打)
```

### ✅ 類型說明 `[xxx]`

| 類型 | 用途說明 |
|------|-----------|
| **`new`** | 新增功能、新模組、新頁面 |
| **`fix`** | 錯誤修正、Bug 修復 |
| **`refactor`** | 程式重構，不影響功能 |
| **`style`** | 介面樣式、排版、格式修正 |
| **`docs`** | 文件、README |
| **`perf`** | 效能優化 or 執行效率改善 |
| **`ci`** | CI/CD、自動化、部署設定相關 |

---

### ✍️ 撰寫原則

1. **摘要用一句話說清楚改了什麼**（不要超過 70 字）。  
2. **多項修改才用點列說明**。  
3. **Commit 前一定要確認能執行、不破壞他人功能。**  
4. **避免無意義訊息**，例如：`update`, `fix again`, `try test`。

## 💡 二、Commit 範例

### 🆕 新增功能
```
[new] 新增股票詳細資訊彈窗

1. 新增 component StockDetailModal.vue
2. 串接股票 API 顯示即時資訊
3. 調整主頁點擊事件觸發流程
```

### 🐞 修正錯誤
```
[fix] 修正報酬率計算錯誤
```

### 🔧 重構
```
[refactor] 優化指標計算模組結構

1. 抽出 RSI 與 MACD 函數至 utils/indicators.py
2. 簡化函數參數傳遞
```

### 🎨 樣式調整
```
[style] 改善首頁卡片 hover 動畫與配色
```

### 📘 文件更新
```
[docs] 更新開發環境安裝與啟動流程
```

## 🌿 三、分支命名規範

### ✅ 命名格式
```
英文名/功能名稱
```

範例：
```
Yii/詳細資料點擊
Eric/圖表優化
```

---

### 🪜 分支建立流程（請務必照順序）

1. **先切回 main 並同步遠端最新版本**
   ```bash
   git checkout main
   git fetch origin
   git pull origin main
   ```

2. **從 main 開新分支**
   ```bash
   git checkout -b Yii/詳細資料點擊
   ```

3. **開發完成後推送**
   ```bash
   git add .
   git commit -m "[new] 新增股票詳細資料彈窗"
   git push origin Yii/詳細資料點擊
   ```

4. **通知負責人進行 merge，不要自行合併到 main。**

## 🚫 四、禁止事項

- [x] Commit 前，請確認修改範圍與影響

---

## 💬 備註

- 所有分支都必須從最新的 **main** 開出來。  
- 開發時請確定「自己知道在做什麼」，不要推上不確定用途的程式碼（套件除外）。  
- 功能完成後，請保留分支並通知合併。  

---
🧩 **保持乾淨、明確、可追蹤的提交歷史，是團隊協作的基本態度。**
